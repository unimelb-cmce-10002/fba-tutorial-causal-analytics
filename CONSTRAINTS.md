# Constraints

This repository is for editing existing Quarto teaching materials, not redesigning them.

## Scope

- Focus on style, formatting, and wording consistency.
- Preserve teaching intent, structure, and exercise flow.

## Do Not Change Without Approval

- Exercise numbering, titles, or sequence
- Analytical content or answers
- Quarto workflow, profiles, or build scripts
- Data, code logic, or YAML beyond formatting fixes

## Editing Rules

- Make the smallest change that resolves the issue.
- Maintain existing voice and level of formality.
- Keep terminology consistent across all sections.
- Preserve Quarto syntax exactly (code blocks, callouts, YAML, includes).
- Do not reformat large sections unnecessarily.

## Reviewability

- When flagging an issue, cite the file path and line number.
- Keep changes narrow and easy to inspect with `git diff`.
- Prefer one file or one issue per edit.
- If a change is ambiguous, leave it and flag it.
